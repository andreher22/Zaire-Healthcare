Propósito del Proyecto
Desarrollar un sistema de salud digital basado en una arquitectura cliente–servidor desacoplada, que permita la gestión de pacientes, el manejo de historial clínico digital y un módulo de apoyo al diagnóstico médico mediante inteligencia artificial, optimizando la atención médica mediante un desarrollo iterativo basado en la metodología Scrum.
Objetivos del Proyecto
Objetivo General
Desarrollar una plataforma digital de salud que integre tecnologías de inteligencia artificial para apoyar el diagnóstico médico, facilitando el acceso a servicios de salud, mejorando la eficiencia en la atención clínica y apoyando la toma de decisiones médicas, siguiendo buenas prácticas de desarrollo de software y la metodología Scrum.
Objetivos Específicos
Implementar un módulo de registro, actualización y seguimiento de pacientes.
Desarrollar un sistema de historial clínico digital con generación de reportes.
Integrar un módulo de análisis preliminar de síntomas basado en IA como apoyo diagnóstico.
Diseñar una arquitectura de software desacoplada, escalable y segura.
Garantizar la usabilidad mediante interfaces móviles intuitivas.
Aplicar Scrum para organizar el trabajo en sprints con entregables funcionales.
Documentar técnica y funcionalmente el sistema desarrollado.
Permitir la movilidad clínica mediante una aplicación móvil distribuida vía APK.
Justificación
La digitalización de los servicios de salud es una necesidad creciente debido al aumento en la demanda de atención médica y la optimización de recursos clínicos. Este proyecto aporta valor al:
Reducir tiempos de registro y consulta de información clínica.
Facilitar el acceso organizado y seguro a historiales médicos digitales.
Proporcionar un apoyo preliminar al diagnóstico, sin sustituir al profesional de la salud.
Introducir el uso de Inteligencia Artificial aplicada a la salud, alineada con tendencias tecnológicas actuales.
Incorporar movilidad clínica mediante una aplicación móvil desarrollada en React Native, permitiendo a los médicos realizar consultas a pie de cama del paciente.
Servir como un caso práctico de aplicación de Scrum en un contexto académico.

El Agente de Diagnóstico por IA
El módulo de IA se implementa como un Servicio de Inferencia integrado en el backend.
Flujo
Entrada: JSON con síntomas enviados desde la app móvil.
Procesamiento: Script Python que ejecuta el modelo de IA.
Salida: Score de probabilidad y recomendación preliminar validada por el médico.
En lugar de un simple script, la IA debe integrarse como un Servicio de Inferencia dentro de Django.
Cómo funciona: El usuario envía sus síntomas desde la app móvil (React Native) a un endpoint específico de Django (ej. /api/diagnostico/).
Procesamiento: Django recibe los datos, los limpia y los pasa a tu modelo de IA (que puede ser un modelo de Machine Learning local con scikit-learn o una llamada a una API de LLM).
Respuesta: El "Agente" devuelve un JSON con la probabilidad del diagnóstico y recomendaciones preliminares.

Módulos necesarios
Se agregó una sección llamada "Especificación de la API" para demostrar profundidad técnica:
Módulo de Seguridad (JWT)
Implementación de autenticación basada en tokens JWT para permitir que la aplicación móvil mantenga la sesión del médico activa entre pantallas, sin solicitar credenciales en cada interacción.
Módulo de Gestión de Pacientes
Exposición de endpoints REST para la administración de pacientes desde la aplicación móvil, tales como:
GET /api/pacientes/ para la consulta del listado de pacientes.
POST /api/pacientes/ para el registro de nuevos pacientes desde el cliente móvil.
Módulo de Historial Clínico
Generación de documentos PDF del historial clínico del paciente desde el backend, mediante un servicio de generación de reportes, los cuales son entregados a la aplicación móvil a través de la API REST para su descarga y previsualización en el dispositivo.

Metodología de Trabajo
El proyecto se desarrolla bajo la metodología ágil Scrum, organizando el trabajo en sprints de duración fija, con entregables funcionales al final de cada iteración. El Product Owner prioriza el backlog de acuerdo con las necesidades del sistema de salud, mientras que el equipo de desarrollo implementa, integra y prueba los módulos del sistema, incluyendo la aplicación móvil, la API REST y el servicio de diagnóstico asistido por inteligencia artificial, promoviendo la mejora continua, la retroalimentación temprana y la validación incremental del producto.

Autenticación de médicos
El sistema deberá permitir al médico iniciar sesión desde la aplicación móvil utilizando credenciales válidas y obtener un token JWT.

El sistema deberá permitir enviar los síntomas del paciente al servicio de diagnóstico asistido mediante el endpoint /api/diagnostico/analizar/.

Estrategia de Conexión y Despliegue
Para visualizar el proyecto en tu teléfono con Expo, necesitamos considerar los siguientes puntos técnicos:
Conexión en Desarrollo (Local):
IP Local: El celular y la computadora deben estar en la misma red Wi-Fi.
Configuración de Django: Debes agregar tu IP local en el archivo settings.py:
ALLOWED_HOSTS = ['tu-ip-local', '127.0.0.1']
CORS: Instalar django-cors-headers para permitir que la app de React Native acepte datos del servidor Django.
Despliegue (Producción):
Backend: Se recomienda usar un servicio como Azure o AWS que soporte SQL Server y Python/Django.
+1
Frontend: Generar el archivo .apk (Android) o .ipa (iOS) usando EAS Build (Expo Application Services).
Seguridad: Implementar HTTPS y autenticación por Tokens (JWT) para proteger los datos médicos sensibles
Infraestructura y Despliegue (Costo Mínimo/Gratis)
Para un proyecto académico, AWS puede ser costoso si te sales de la capa gratuita. Por ello se usarán estás opciones:
Backend (Django): Render o Railway: Tienen planes gratuitos o de "pago por uso" muy bajos (aprox. $5 USD/mes) que son más fáciles de configurar que AWS para estudiantes.
Base de Datos (SQL Server): * Si es obligatorio usar SQL Server, puedes usar la versión Express en un servidor local o buscar el nivel gratuito de Azure SQL Database (que suele dar 12 meses gratis a estudiantes).
Frontend (App Móvil):
Expo Go: Para la fase de desarrollo y revisión académica, no necesitas desplegar nada. Los profesores pueden escanear un código QR y ver la app en sus teléfonos al instante.
APK (Android): Se genera usando EAS Build (eas build -p android --profile preview). Expo te permite hacer builds en la nube de forma gratuita (con cola de espera).
En caso de usar AWS::
App Runner: Para desplegar el contenedor de Django de forma sencilla.
RDS (SQL Server): Ojo, es caro. Solo úsalo si tienes créditos de estudiante de AWS Educate.
S3: Para guardar reportes clínicos o imágenes si el proyecto lo requiere.

Conclusión
El proyecto se desarrolla bajo una arquitectura cliente–servidor desacoplada, basada en una API REST construida con Django REST Framework y un cliente móvil desarrollado en React Native.
